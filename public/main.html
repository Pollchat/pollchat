Pollchat

We aim to have a system that will allow users to log in via an Oauth provider (likely Google/Facebook) to create and participate in polls with other users. This will be accomplished by the user of websockets connecting to a Go web server. We will process the new question and then give the user a link to share with their friends in order to participate in the voting. Alongside the voting we aim to implement a simple chat window that is linked to the vote in order for people to let others know they have completed it etc.

Created by Tom Townsend & Laura McCormack

The role of the Login page is to allow users to log in to the poll application. This is done done using two input fields and an input button.

The role of the index page is to show the user a automatically updating list of recent polls whilst also providing the facility to create a new poll. This is achieved by the use of a form for inputting a new poll with answers - this has five input fields and an input submit button. The right hand side contains divs with recent polls - showing the poll question and the most popular answer so far.

The poll page allows the user to see the status of a poll and participate in a chat about that poll. This is achieved by two sections on the page. the left hand side of the page has the Poll question and answers and a graph below. The right hand side of the screen has an input text field and a input submit button for users to input comments into the chat bar. Below the input text is the chat bar - a list of users against their messages using span objects to get the correct layout for the chat section.
